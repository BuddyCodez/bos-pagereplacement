Page Replacement Algorithm

What is Page Replacement Algorithm?

In an operating system that uses paging for memory management, a page replacement algorithm is needed to decide which page needs to be replaced when a new page comes in.

Comparision between FIFO and LRU Algorithm

First In First Out

In this algorithm, operating system keeps track of all pages in the memory in a queue, oldest page is in the front of the queue. When a page needs to be replaced page in the front of the queue is selected for removal.

Least Recently Used

this algorithm is based on the strategy that whenever a page fault occurs, the least recently used page will be replaced with a new page. So, the page not utilized for the longest time in the memory gets replaced.

Conclusion for FIFO:

FIFO page replacement algorithm is definitely not the best page replacement algorithm to use practically. When the number of incoming pages is less, and a user is looking for a simple approach, FIFO might be a reasonable choice.

Conclusion for LRU:

The LRU page replacement algorithm replaces the least recently used page in the memory with a new page. The LRU page replacement algorithm is based on the assumption that among all pages in the memory, the least recently used page will not be used for a long time
